# Execution States

> You set a state in `03_DAILY/today.md`. The state tells every AI how to behave.
> This protects your energy and keeps the system aligned with how you're actually operating.

---

| State | When to use it | What AI should do | Tasks allowed |
|-------|----------------|-------------------|---------------|
| 🔴 **Focus** | Deep work, one thing | Be quiet and precise. Short answers. No new ideas. | P0 only |
| ⚡ **Execution** | Clearing the list | Be action-oriented. Drive tasks to done. | P0 + P1 |
| 🌱 **Growth** | Learning, exploring | Offer ideas, options, creative input. | P0–P2 |
| 🗺️ **Planning** | Strategy & review | Help organize and decide. Light execution. | Planning + light P1 |
| 🔋 **Recovery** | Low energy, rest | Be brief. Only quick wins. | Light only (< 30 min) |

## How it works

1. Set the state at the top of `today.md`.
2. Any AI reads it before suggesting or doing anything.
3. If a request conflicts with the state, the AI gently flags it before proceeding.

## Default

If `today.md` has no state set, assume **⚡ Execution**, keep responses tight, and ask for your state.

---

*The state always wins over the AI's own sense of what "should" happen next.*
