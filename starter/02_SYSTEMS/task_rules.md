# Task Rules

> How tasks and projects are defined, prioritized, and tracked.

---

## Definitions

- **Task** — a single action you can finish in one sitting.
- **Project** — anything requiring more than one task. Lives as a folder in `04_PROJECTS/`.
- **Domain** — an ongoing life/work area (no end date). Lives in `01_BRAIN/domains/`.

## Priority ladder

| Level | Meaning | Response time |
|-------|---------|---------------|
| **P0** | Critical | Act immediately |
| **P1** | High | Within 24 hours |
| **P2** | Standard | Within 3–5 days |
| **P3** | Low | When bandwidth exists |
| **Someday** | Backlog | Review weekly |

**Rules:**
- Never more than **3 P0s** at once. If everything is P0, reprioritize.
- Every task has one priority and ideally one next action.
- A task with no clear next action is a *project* — break it down.

## Task format (in today.md)

```
- [ ] (P1) Call the bank about the wire   →  #finances
- [ ] (P2) Draft Sunday video script      →  #content
```

Checkbox = status. `(P#)` = priority. `#tag` = domain.

## Moving tasks

- Finished → check it off; it stays as history.
- Not done today → carry forward to the next `today.md`.
- No longer relevant → move to Someday (or delete with permission).

---

*Keep it simple. The system should reduce friction, not add it.*
